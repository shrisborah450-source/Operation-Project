## 1. Supply Chain & Demand-Supply Management

In service operations, the supply is your team's capacity. The demand is incoming tickets, cases, and approvals. My job is to make sure supply never falls too far behind demand.

| Step | What I Did | Result |
|------|-----------|--------|
| Demand tracking | Monitored daily ticket inflow and flagged spikes early | Prevented 3 potential SLA breach events |
| Capacity mapping | Tracked each team member's processing speed | Realistic workload distribution |
| Priority tagging | Tagged tickets as urgent, standard, or low-priority at intake | Urgent resolution time down 30% |
| Backlog monitoring | Tracked age of open tickets daily | Consistent SLA adherence across all workflows |
| Cross-team coordination | Flagged supply gaps before demand exceeded capacity | Prevented downstream failures |

> **Insight:** The problem almost never announces itself. I got into the habit of checking queue depth every morning, not at the end of the day.

## 3. Process Improvement

I do not wait to be asked to find problems. At Capslock, I identified and fixed 3 recurring bottlenecks.

**Bottleneck 1 — Ticket Routing Delays**
- Problem: Urgent tickets processed in the same order as low-priority ones
- Root Cause: No differentiation at intake
- Fix: Designed a priority-tagging system — urgent items go to a fast-track queue immediately
- **Result: Urgent resolution time reduced by 30%**

**Bottleneck 2 — KYC Rework Due to Unclear Criteria**
- Problem: Documents sent back because validators had different standards
- Root Cause: SOP had gaps — edge cases not documented
- Fix: Rewrote KYC validation procedure with explicit criteria for all 5 document types
- **Result: Zero regulatory breaches, measurable drop in rework**

**Bottleneck 3 — Cross-team Escalations Taking Too Long**
- Problem: Tickets needing Tech or Compliance input sat with no clear owner
- Root Cause: No defined escalation protocol
- Fix: Built an escalation matrix with named owners, SLA per level, daily blocker check-in
- **Result: Cross-team resolution improved, downstream SLA failures reduced**

> **Insight:** Every bottleneck had the same root cause — a process designed for a simpler environment that was never updated as the team grew. The fix is rarely more people. The fix is a clearer process.
